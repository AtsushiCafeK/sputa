# Google Apps Script セットアップガイド

## 概要

このGASスクリプトは、Googleフォームの回答をスプレッドシートで自動管理し、PHPバックエンドと連携するものです。

---

## ファイル構成

```
main.gs           - メインスクリプト（トリガー関数）
config.gs         - 設定・定数
api-client.gs     - PHP API通信クライアント
```

---

## セットアップ手順

### ステップ 1: Google Apps Script プロジェクト作成

1. **スプレッドシートを開く**
   - 作成済みの回答スプレッドシート（フォーム連携済み）を開く

2. **スクリプトエディタを開く**
   - メニュー: `拡張機能` → `Apps Script`
   - または: 右上の `< > Apps Script` ボタン

3. **デフォルトファイルをクリア**
   - `Code.gs` をすべて削除して空にする

4. **ファイルを追加・配置**
   - `main.gs` をコピー&ペースト
   - `+ ファイルを作成` → `スクリプト`
     - `config.gs` を作成してコピー&ペースト
     - `api-client.gs` を作成してコピー&ペースト

---

### ステップ 2: APIキー設定

1. **PHPバックエンドのAPIキーを取得**
   - さくらサーバ `/app/api/.env` で `API_KEY` の値を確認
   - 例: `API_KEY=c1a2b3d4e5f6g7h8...`

2. **GASでAPIキーを設定**
   - GAS エディタで関数 `applySettings()` を選択
   - ▶ 実行ボタンをクリック
   - 認可ダイアログが出たら `続行` → アカウント選択 → `許可`

3. **スクリプトプロパティを手動修正（推奨）**
   - メニュー: `プロジェクト設定` → `スクリプトプロパティ` をクリック
   - テーブルを確認して `API_KEY` の値を **実際のキーに置き換える**
   - 例: `YOUR_API_KEY_HERE` → `c1a2b3d4e5f6g7h8...`

4. **設定テスト**
   - 関数 `testApiKeySettings()` を実行
   - ログに「✓ APIキーが設定済みです」と表示されれば成功

---

### ステップ 3: トリガー設定

#### トリガー A: フォーム送信時

1. GAS エディタ左側の `トリガー` アイコンをクリック
2. `トリガーを作成` ボタン
3. 以下を設定:
   ```
   実行する関数を選択: onFormSubmit
   実行するデプロイを選択: Head
   イベントのソースを選択: フォーム
   イベントの種類を選択: フォーム送信時
   ```
4. `保存` をクリック
5. 認可ダイアログ → `続行` → アカウント選択 → `許可`

#### トリガー B: スプレッドシート編集時

1. `トリガーを作成` ボタン
2. 以下を設定:
   ```
   実行する関数を選択: onEdit
   実行するデプロイを選択: Head
   イベントのソースを選択: スプレッドシート
   イベントの種類を選択: 編集時
   ```
3. `保存` をクリック
4. 認可ダイアログ → `続行` → アカウント選択 → `許可`

#### トリガー設定確認

- `トリガー` ページで以下の2つが表示されていること:
  ```
  onFormSubmit    | フォーム | フォーム送信時
  onEdit          | スプレッドシート | 編集時
  ```

---

### ステップ 4: テスト実行

#### テスト A: API接続確認

1. GAS エディタで関数 `runTestApiCall()` を選択
2. ▶ 実行ボタンをクリック
3. 実行ログを確認（GAS エディタの下部）
   ```
   === テストAPI呼び出し開始 ===
   → PHPに送信: create (...)
   ← PHPからのレスポンス (HTTP 200)
     本文: {"success":true,...}
   ✓ API呼び出し成功
   === テストAPI呼び出し終了 ===
   ```
   - `✓ API呼び出し成功` が表示されれば **接続成功**
   - エラーが出た場合は、APIキーやURLを確認

#### テスト B: フォーム送信テスト

1. Googleフォームを開く
2. テスト回答を送信
3. スプレッドシート確認
   - G列（ステータス）が `未着手` に自動設定されているか
   - H列（進捗）が `0` に自動設定されているか
4. GAS 実行ログ確認
   - `✓ フォーム送信処理完了` というログが出ているか

#### テスト C: スプレッドシート編集トリガー

1. スプレッドシートで任意の行のG列（ステータス）を `対応中` に変更
2. GAS 実行ログ確認
   - `✓ スプレッドシート更新処理完了` というログが出ているか

## トラブルシューティング

### エラー: "Unauthorized (401): APIキーが正しくありません"

**原因**: APIキーがPHPバックエンドに登録されたものと一致していない

**対処法**:
1. PHPバックエンド `/app/api/.env` のAPIキーを確認
2. GASのスクリプトプロパティで設定を修正
3. `testApiKeySettings()` を実行して確認

---

### エラー: "API_KEY が設定されていません"

**原因**: GASのスクリプトプロパティにAPIキーが設定されていない

**対処法**:
1. `applySettings()` を実行
2. プロジェクト設定 → スクリプトプロパティで値を確認・修正

---

### エラー: "Invalid JSON response"

**原因**: PHPバックエンドが正常なJSONを返していない

**対処法**:
1. PHPバックエンドのURL (`API_BASE_URL`) が正しいか確認
2. PHPサーバログを確認（エラーメッセージがないか）
3. 次フェーズ（PHP API実装）で詳細を確認

---

### トリガーが実行されない

**原因**: トリガー設定が正しくない、またはスプレッドシートが共有フォルダにある

**対処法**:
1. トリガー設定を再確認
2. フォームとスプレッドシートのオーナーが同じアカウントか確認
3. スプレッドシートのアクセス権限を確認

---

## ログの確認方法

1. GAS エディタ上部の `ログ` をクリック
2. または Ctrl+Enter（実行後）に自動表示

### ログレベル

```
✓ = 正常完了
✗ = エラー発生
→ = PHPへ送信
← = PHPからレスポンス受信
```

---

## 本番環境での注意事項

- **APIキーは秘密にする**: スクリプトプロパティに保存し、コードに直書きしない
- **ログの定期削除**: GASログが容量制限に達しないよう定期的に削除
- **エラーログシート**: 「エラーログ」シートが自動作成され、エラーが記録される
